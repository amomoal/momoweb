# Web Support デザイン仕様

## 1. 横幅 max-width
- 基本はスマホ優先の 1カラム構成
- コンテンツ全体の最大幅は `640px`
- 画面が広い場合も中央寄せで表示する
- 本文、カード一覧、各ページ本文は同じ横幅ルールで統一する

### ルール
- `width: 100%`
- `max-width: 640px`
- `margin: 0 auto`

---

## 2. 余白の基準
- 余白は詰めすぎず、読みやすさ優先
- 左右余白は常に一定にする
- セクション間は広めに取って、情報の塊を分ける
- カード内余白は本文より少し広めにする

### 基準値
- 画面左右余白: `16px`
- セクション上下余白: `56px`
- セクション見出し下余白: `20px`
- カード内余白: `16px`
- 要素同士の縦間隔: `12px`
- 小さめ余白: `8px`

### 余白ルール
- 迷ったら `8 / 12 / 16 / 20 / 32 / 56` の段階で使う
- 独自の中途半端な値はなるべく増やさない

---

## 3. 見出しサイズ
- 見出しは3段階で統一する
- 本文よりしっかり差をつけるが、大げさにはしない
- スマホでも読みやすいサイズを優先する

### サイズ
- ページ大見出し: `32px`
- セクション見出し: `24px`
- 小見出し: `18px`
- 本文: `16px`
- 補足テキスト: `14px`

### 行間
- 大見出し: `1.25`
- セクション見出し: `1.35`
- 本文: `1.7`
- 補足: `1.6`

### 太さ
- 大見出し: `700`
- セクション見出し: `700`
- 小見出し: `600`
- 本文: `400`

---

## 4. カードのデザイン
- アプリ紹介はカード形式で統一する
- やわらかく上品な印象を優先する
- 影は薄く、境界線と角丸で整える
- カードは1カラムで縦並びにする

### デザイン
- 背景色: `var(--card)`
- 枠線: `1px solid var(--line)`
- 角丸: `20px`
- 内側余白: `16px`
- 外側の縦間隔: `16px`
- 影: ごく薄め

### 影
- `box-shadow: 0 4px 14px rgba(43, 36, 31, 0.08);`

### カード内構成
- アプリアイコン
- アプリ名
- 短い説明文
- 詳細ページへのボタン
- App Storeリンクがある場合は別ボタンで配置

---

## 5. ボタンのデザイン
- ボタンは押しやすさ優先
- 小さすぎるボタンは禁止
- カードと世界観を揃えて、丸みを持たせる
- 主ボタンと副ボタンの2種類までに絞る

### 主ボタン
- 背景色: `var(--accent)`
- 文字色: `#ffffff`
- 高さ: `44px`
- 左右余白: `16px`
- 角丸: `999px`
- 太さ: `600`

### 副ボタン
- 背景色: `transparent`
- 文字色: `var(--accent)`
- 枠線: `1px solid var(--line)`
- 高さ: `44px`
- 左右余白: `16px`
- 角丸: `999px`

### ボタンルール
- 1画面内のボタン種類は増やしすぎない
- hover演出は軽め
- スマホで押しやすいサイズを維持する

---

## 6. 色を3〜4色に絞る
- 色数は増やしすぎず、上品で統一感ある構成にする
- ベースはアイボリー、紙色、ラベンダーグレー、濃色文字でまとめる
- 強い差し色は使わず、落ち着いたアクセント色を使う

### 使用カラー
- 背景色: `#FFF8F2`
- セクション背景: `#F3ECE8`
- カード背景: `#EEE9F0`
- 本文色: `#2B241F`
- アクセント色: `#6E6A9A`
- 線色: `#D8CEC6`

### CSS変数
```css
:root {
  --bg: #FFF8F2;
  --surface: #F3ECE8;
  --card: #EEE9F0;
  --text: #2B241F;
  --accent: #6E6A9A;
  --line: #D8CEC6;
}

色ルール
ページ背景は --bg
セクション背景は --surface
カード背景は --card
本文と見出しの基本文字は --text
リンク、主ボタン、強調は --accent
枠線、区切り線は --line

7. レイアウト方針
スマホ版を基準に作る
1カラム固定
PCでは中央寄せ表示のみ対応し、専用レイアウトは作らない
レスポンシブ対応は最小限にとどめる
方針
スマホで見やすいことを最優先
PCでは壊れないことを優先
まずは情報整理を優先し、凝った配置は後回しにする
